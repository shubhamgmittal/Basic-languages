Bash Setup Guide for Windows

Required Software

Before setting up Bash, ensure you have the following installed on your Windows system:


  • Windows: Version 10 or 11 (64-bit recommended)

  • Code Editor: Visual Studio Code (recommended)

  • Bash Environment: WSL (Windows Subsystem for Linux), Git Bash, or MSYS2

  • Administrator Access: For installing software and enabling features

  • Git: For version control and Git Bash (optional but recommended)

Steps of Installations


Step 1: Enable WSL (Recommended for Full Bash Experience)


  1. Open PowerShell as Administrator

  2. Enable WSL feature:

    dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart

  3. Enable Virtual Machine Platform:

    dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

  4. Restart your computer

  5. Install WSL2 kernel update (download from Microsoft website)

  6. Set WSL2 as default:

    wsl --set-default-version 2

  7. Install a Linux distribution (e.g., Ubuntu):

    wsl --install -d Ubuntu


Step 2: Alternative - Install Git Bash


  1. Download Git for Windows from git-scm.com

  2. Run the installer

  3. Select "Use Git from Git Bash only" during installation

  4. Complete the installation

  5. Launch Git Bash from Start menu


Step 3: Install VS Code


  1. Download VS Code installer from the official website

  2. Run the installer executable

  3. Click "Next" and accept the license agreement

  4. Choose installation location (default is fine)

  5. Complete the installation and launch VS Code


Step 4: Configure VS Code for Bash


  1. Open VS Code

  2. Press Ctrl+Shift+P to open Command Palette

  3. Type "Terminal: Select Default Profile" and select WSL or Git Bash

  4. Open integrated terminal: Ctrl+`

  5. Verify Bash is working: echo "Hello, Bash!"


Step 5: Install VS Code Extensions


  1. Open Visual Studio Code

  2. Press Ctrl+Shift+X to open the Extensions panel

  3. Search for "Bash" and install relevant extensions

  4. Click "Install" for each extension

  5. Reload VS Code by pressing Ctrl+Shift+P and selecting "Developer: Reload Window"


Step 6: Create Your First Bash Script


  1. Open VS Code and create a new file: hello.sh

  2. Add shebang: #!/bin/bash

  3. Add code: echo "Hello, Bash!"

  4. Save the file

  5. Make it executable: chmod +x hello.sh (in WSL/Git Bash)

  6. Run the script: ./hello.sh

Basic Lines of Code for Beginners Sections

Here are simple Bash scripts to get you started:


Hello World Script


#!/bin/bash
echo "Hello, World!"

Variables


#!/bin/bash
name="Alice"
age=25
echo "Name: $name"
echo "Age: $age"

User Input


#!/bin/bash
echo "What is your name?"
read name
echo "Hello, $name!"

Conditional Statements


#!/bin/bash
echo "Enter a number:"
read num
if [ $num -gt 10 ]; then
    echo "Number is greater than 10"
elif [ $num -eq 10 ]; then
    echo "Number is equal to 10"
else
    echo "Number is less than 10"
fi

Loops


#!/bin/bash
# For loop
for i in {1..5}; do
    echo "Number: $i"
done

# While loop
count=1
while [ $count -le 5 ]; do
    echo "Count: $count"
    ((count++))
done

Functions


#!/bin/bash
greet() {
    echo "Hello, $1!"
}

greet "World"
greet "Bash"

File Operations


#!/bin/bash
# Create a file
echo "This is a test file" > test.txt

# Read a file
cat test.txt

# Check if file exists
if [ -f test.txt ]; then
    echo "File exists"
else
    echo "File does not exist"
fi

# Remove file
rm test.txt

Troubleshooting Common Issues


Issue 1: WSL Installation Fails

Problem: WSL installation fails with error messages

Solution: Ensure virtualization is enabled in BIOS, run as Administrator


Issue 2: Bash Command Not Found

Problem: 'bash: command not found' in terminal

Solution: Ensure WSL or Git Bash is properly installed and in PATH


Issue 3: Permission Denied

Problem: 'Permission denied' when running scripts

Solution: Make script executable with chmod +x script.sh


Issue 4: Extensions Not Working

Problem: Bash extensions not providing syntax highlighting

Solution: Ensure file has .sh extension and correct shebang


Issue 5: Terminal Not Opening in VS Code

Problem: Cannot open integrated terminal

Solution: Configure default terminal profile in VS Code settings

Useful Commands for Bash Development


  • bash --version - Check Bash version

  • which bash - Find Bash location

  • chmod +x file.sh - Make script executable

  • ./script.sh - Run a Bash script

  • echo $PATH - Show PATH environment variable

  • ls -la - List files with details

  • pwd - Print working directory

  • cd directory - Change directory

  • mkdir dirname - Create directory

  • rm filename - Remove file

  • cat filename - Display file contents

  • grep pattern file - Search for pattern in file
